RESUMO EXECUTIVO

1. ESTRUTURA PRINCIPAL
   Organizada em: HTML/CSS/JS segregados
   Módulos: shared, character, compendium, auth
   Dados: Centralizados no Supabase

2. FICHAS (2 sistemas)
   - D&D 5e: 735 linhas JS + 1061 linhas HTML
   - M&M 3e: 618 linhas JS + interface completa
   - Listagem: my-characters.js (132 linhas)

3. CRIAÇÃO DE FICHAS
   - Via botão "Nova Ficha [Tipo]"
   - Limite: 5 fichas por usuário
   - Verifica limite antes de criar
   - Redireciona para editor apropriado

4. ARMAZENAMENTO (3 camadas)
   - Camada 1: Supabase PostgreSQL (primária)
   - Camada 2: SessionStorage (ID ativo)
   - Camada 3: LocalStorage (fallback)
   - Sincronização automática a cada 1500ms

5. EXECUÇÃO AUTOMÁTICA
   - requireAuth() bloqueia acesso não autenticado
   - DOMContentLoaded dispara loadCharacter()
   - carregarFichas() executa ao carregar my-characters.js
   - autoSave() monitora mudanças continuamente
